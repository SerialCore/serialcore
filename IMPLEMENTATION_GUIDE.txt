Self-Organizing Neural Network (SONN) Design Summary
Flexible Connections · Memory-Pool Architecture · C Language Implementation
This document consolidates all design decisions, code patterns, and recommendations from the conversation for easy agent parsing and implementation.
1. Project Context & Input Dimension Guidelines
The network is a self-organizing neural network with dynamic, flexible connections (not fixed layers, but individual neurons as basic units).
Target input dimensions (configurable at initialization):

64, 128, 256, 512, 1024 (powers of 2 preferred for cache alignment and hardware efficiency)

Typical ranges observed across domains:

Production-grade FC networks: up to thousands–tens of thousands
Scientific (NNP): 50–500
Financial prediction: 20–200
Neuromorphic / brain-inspired: 64–512 (fan-in)

Recommended starting values: 256 or 512 for most research/prototyping.
2. Core Design Principles

Neurons are the fundamental unit (not layers).
Connections are fully dynamic (add / remove / age / strengthen).
Memory is pre-allocated once at creation; runtime operations avoid frequent malloc/free.
Use memory pool + index-based management + adjacency list for topology.
Support growth mechanisms (e.g., Growing Neural Gas style).

3. Memory Pool & Data Structures (C Language)
Recommended structs (AOS / SOA hybrid):
Ctypedef struct {
    float* weights;        // pointer into unified weight array or SOA block
    int    id;
    int    active;
    float  error;          // for GNG-style error-driven growth
    // other neuron-specific fields (position, activation type, etc.)
} Neuron;

typedef struct {
    int   from;
    int   to;
    float strength;
    float age;
} Edge;

typedef struct {
    Neuron* neurons;
    Edge*   edges;
    int*    free_list;          // indices of inactive neurons
    int     free_count;
    int     current_neurons;
    int     max_neurons;
    int     input_dim;
    // Adjacency list or edge list for flexible connections
} Network;
Network creation (memory pool initialization):
CNetwork* create_network(int max_n, int dim) {
    Network* net = (Network*)malloc(sizeof(Network));
    net->max_neurons = max_n;
    net->input_dim = dim;
    net->current_neurons = 0;

    net->neurons = (Neuron*)calloc(max_n, sizeof(Neuron));
    net->edges   = (Edge*)calloc(max_n * 50, sizeof(Edge));   // assume avg degree ~50

    // Free list initialization
    net->free_list = (int*)malloc(max_n * sizeof(int));
    for (int i = 0; i < max_n; i++) net->free_list[i] = i;
    net->free_count = max_n;

    return net;
}
Runtime operations (no frequent malloc/free):

add_neuron() → take slot from free_list, initialize weights, increment current count.
remove_neuron(int id) → mark inactive and return to free list.
add_edge(int from, int to), remove_edge() → modify adjacency list or edge array only.

Performance tip: Prefer SOA layout (all weights in one contiguous array) for better cache behavior when input_dim is 256/512/1024.
4. Serialization & Model I/O (Standardized Formats)
Recommended combination:

Weights → Safetensors (or raw binary tensor dump) for zero-copy efficiency.
Topology + Metadata → Protocol Buffers (protobuf) for structured, versioned, cross-language storage.

Protobuf schema example:
protomessage Neuron {
    int32 id           = 1;
    int32 input_dim    = 2;
    string activation  = 3;     // "linear", "sigmoid", ...
    bytes weights      = 4;     // or external safetensors offset
    float error        = 5;
    int32 active       = 6;
}

message Edge {
    int32 from     = 1;
    int32 to       = 2;
    float strength = 3;
    float age      = 4;
}

message Network {
    int32 max_neurons     = 1;
    int32 current_neurons = 2;
    int32 input_dim       = 3;
    repeated Neuron neurons = 4;
    repeated Edge edges     = 5;
    map<string, string> metadata = 6;   // version, training info, etc.
}
Alternative / fallback formats:

JSON / MessagePack (fast prototyping & debugging)
HDF5 (scientific computing, large arrays + attributes)
Custom binary header + memory-mapped weight block (maximum performance)

Save/Load workflow:

Serialize topology + metadata via protobuf.
Dump weights to Safetensors or raw binary (memory mirror).
On load: read protobuf → allocate/fill memory pool → load weights.

5. Quick Start Recommendations

Start with max_neurons = 1000–10000, input_dim = 256 or 512.
Use adjacency list for connections.
Implement GNG-style growth rules (error-driven neuron/edge addition + aging).
Add version field in metadata for future compatibility.

This architecture provides:

High flexibility of connections
Excellent runtime performance
Clean serialization
Easy experimentation across different input dimensions

Ready for implementation or further refinement.
